SSH Connection Guide

Connecting to lab servers via SSH — the primary method for remote development.

Method 1: VS Code Remote-SSH (Recommended)

The most convenient approach — edit server files directly in VS Code with full IDE support.

# 1. Install "Remote - SSH" extension in VS Code
# 2. Open Command Palette (Ctrl+Shift+P)
# 3. Select "Remote-SSH: Connect to Host..."
# 4. Enter the SSH command:
ssh username@server-ip

Method 2: Terminal SSH

Direct terminal connection for running commands and scripts on the server.

ssh username@server-ip-address

After connection, you can navigate the server file system, run Python scripts, manage GPU resources, and monitor training jobs — all from the terminal.

Server Network Constraints

Lab servers cannot access foreign networks directly. This affects tools that pull data from abroad.

Affected tools:

conda huggingface pip (foreign packages)

Workarounds & Mirror Configuration

Two options when the server cannot reach foreign sources directly:

Option 1: Configure Domestic Mirrors

# Conda: add Tsinghua mirror
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --set show_channel_urls yes

# pip: use Tsinghua mirror
pip install package-name -i https://pypi.tuna.tsinghua.edu.cn/simple

# Hugging Face: use hf-mirror.com
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download model-name --local-dir ./models

Option 2: Download Locally, then Upload via WinSCP

# Step 1: Download model weights on your local machine
# (with access to foreign networks)

huggingface-cli download meta-llama/Llama-2-7b-hf ./llama-2-7b

# Step 2: Upload the folder to the server via WinSCP
# Drag and drop in the WinSCP GUI, or use SCP:

scp -r ./llama-2-7b username@server-ip:/path/to/projects/models/

# Step 3: Point your code to the local path on the server
export MODEL_PATH=/path/to/projects/models/llama-2-7b
